feat(oauth): implement complete Google OAuth 2.0 flow with handlers and comprehensive tests

This commit completes Milestone 5 by implementing the full OAuth 2.0 authentication flow
with Google as the first provider. The implementation includes both /authorize and 
/callback endpoints, along with automatic user creation, session management, and 
comprehensive test coverage.

HANDLERS IMPLEMENTATION (handlers/auth.go):
- OAuthAuthorize handler: Initiates OAuth flow with provider validation
  * Validates required parameters (provider, app_id, redirect_uri)
  * Checks if provider is valid and enabled for the application
  * Generates secure state token using crypto/rand (32 bytes hex-encoded)
  * Stores OAuth state in Redis with 5-minute TTL
  * Constructs provider-specific authorization URL
  * Redirects user to OAuth provider (Google)

- OAuthCallback handler: Processes OAuth provider callback
  * Validates code and state parameters
  * Retrieves and validates OAuth state from Redis (single-use)
  * Exchanges authorization code for access and refresh tokens
  * Fetches user information from OAuth provider
  * Creates or updates user and identity records
  * Generates JWT token with user, app, and session claims
  * Creates session record with token hash, IP address, and user agent
  * Redirects to client application with JWT token

- Helper functions:
  * generateStateToken(): Generates cryptographically secure random state tokens
  * findOrCreateUser(): Handles user and identity creation/update logic
    - Searches for existing identity by provider and provider_user_id
    - Updates access/refresh tokens if user exists
    - Creates new user and identity records if not found
    - Returns user UUID for session creation

MAIN APPLICATION UPDATES (main.go):
- Registered OAuth routes in public API group:
  * GET /api/v1/oauth/authorize - Initiates OAuth flow
  * GET /api/v1/oauth/callback/:provider - Handles provider callback
