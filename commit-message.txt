feat(middleware): implement API key authentication middleware

Implement comprehensive API key validation middleware for protecting admin routes and
application-scoped endpoints. This middleware enables secure authentication for client
applications using API keys stored in the database.

Core Features Implemented:

1. RequireAPIKey Middleware
   - Validates API key from X-API-Key header
   - Queries applications table to verify key existence
   - Loads application context into Fiber locals for downstream handlers
   - Returns proper HTTP status codes (401 for auth failures, 500 for DB errors)
   - Implements clear error messages for debugging

2. Helper Functions
   - GetApplication(): Retrieves full Application model from context
   - GetApplicationID(): Extracts only the UUID for performance
   - Type-safe context access with proper error handling

3. Integration with Main Application
   - Added /api/v1/admin route group with RequireAPIKey middleware
   - Implemented test endpoint to verify middleware functionality
   - Updated imports to include middleware package

Technical Implementation Details:

Database Query:
- Direct pgx query for performance (no ORM overhead)
- Efficient single-query application lookup by api_key
- Handles pgx.ErrNoRows for non-existent keys
- Proper error propagation for database failures

Context Management:
- Uses Fiber's c.Locals() for request-scoped storage
- Type-safe retrieval with runtime type checking
- Clear error messages when context is invalid

Security Considerations:
- No API keys logged in error messages
- Clear separation of auth failures vs system errors
- Proper HTTP status codes for different failure modes
- Database connection handled safely with context.Background()
