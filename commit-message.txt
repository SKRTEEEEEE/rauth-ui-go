feat(middleware): implement JWT authentication middleware with comprehensive testing

Implement RequireAuth middleware for JWT token validation with database session verification.
This middleware protects endpoints by validating JWT tokens and ensuring corresponding
sessions exist in the database before allowing access to protected resources.

Changes:
- Implemented RequireAuth middleware function to validate JWT tokens from Authorization header
- Added session validation against database with automatic expiration checking
- Implemented GetJWTClaims helper to retrieve JWT claims from request context
- Implemented GetSession helper to retrieve session data from request context
- Added automatic last_used_at timestamp updates for active sessions
- Created comprehensive unit tests covering all error scenarios and edge cases
- Created integration tests with full database interaction scenarios
- Validated Bearer token format (rejects malformed Authorization headers)
- Implemented proper error responses for unauthorized access attempts
- Added context propagation for downstream handlers to access user/session data

Test Coverage:
- Unit tests: 15 test cases covering middleware behavior without database
  - Missing Authorization header validation
  - Invalid token format detection (5 scenarios)
  - Malformed JWT token rejection
  - Context helper functions (GetJWTClaims, GetSession)
  - Invalid context data type handling
- Integration tests: 7 test cases with database interaction

  - Valid token with non-existent session (rejection)
  - Valid token with expired session (rejection)

  - Session last_used_at timestamp update verification
  - GetJWTClaims integration with real middleware chain
  - GetSession integration with real middleware chain

Security Features:
- Bearer token extraction with strict format validation
- JWT signature verification using configured secret
- Token hash validation against database records
- Session expiration enforcement at database level (expires_at > NOW())
- Automatic session activity tracking (last_used_at updates)
- Protection against session hijacking via token hash matching
- Proper error messages without leaking sensitive information

Implementation Details:
- Uses utils.ValidateJWT for token parsing and claim extraction
- Uses utils.HashToken for secure token hash generation (SHA-256)
- Queries sessions table with compound conditions (id, token_hash, expires_at)
- Stores validated claims and session in Fiber context for handler access
- Returns appropriate HTTP status codes (401 for auth failures, 500 for DB errors)
- Ignores errors on last_used_at update to avoid blocking valid requests
