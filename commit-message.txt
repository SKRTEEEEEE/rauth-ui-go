## Database Schema Details

**applications**: Client applications using the auth service
- UUID primary key with auto-generation
- Unique API key for authentication
- Text arrays for allowed redirect URIs and CORS origins

**oauth_providers**: OAuth provider configuration per application
- Supports: google, github, facebook, microsoft
- Unique constraint on (app_id, provider) combination
- Enabled flag for easy on/off toggling

**users**: End users of client applications
- Linked to applications via foreign key
- Unique email per application
- Avatar URL support for Azure Blob Storage integration

**identities**: OAuth identities linked to users
- Stores provider user IDs and tokens (encrypted)
- Unique constraints prevent duplicate provider connections
- Supports multiple OAuth identities per user

**sessions**: Active user sessions
- JWT token hash for validation
- Tracks IP address and user agent
- Automatic expiration handling

## Performance Optimizations
- 9 strategic indexes for query optimization

- Efficient CASCADE deletes reduce orphaned records
