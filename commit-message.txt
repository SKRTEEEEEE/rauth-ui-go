**Key Design Decisions:**

1. Pointer vs Value Types:
   - Pointers (*string, *time.Time) for nullable database fields
   - Values (string, time.Time, bool) for NOT NULL fields
   - Arrays ([]string) for PostgreSQL array columns

2. JSON Tags:
   - json:"field_name" for standard serialization
   - json:"-" for sensitive fields (tokens, hashes)
   - json:"field,omitempty" for optional request fields

3. Database Tags:
   - db:"column_name" for pgx scanning
   - Matches exact column names from migrations.sql

4. Type Safety:
   - uuid.UUID for all ID fields (not strings)
   - time.Time for all timestamp fields
   - Provider constants to prevent typos

5. Security:
   - AccessToken, RefreshToken, TokenHash never exposed in JSON
   - Proper use of omitempty for partial updates
   - Email marked as omitempty in JWT claims
